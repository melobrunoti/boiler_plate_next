### Pasta Api

Destinada a nossas rotas de api e nossas queries ( React querie ).

### Pasta App

Esta pasta é a principal característica do Next.js 13/14 onde criamos nossas rotas nomeando o arquivo como page.tsx.
Onde também é possível criar grupos de pastas criando a mesma com o nome entre parênteses, ex: (nome_do_grupo), que nos permite aplicar diferentes layouts para cada grupo, exemplo componentes como Header e Sidebar só serão exibidos para usuários logados então os adicionamos apenas ao layout do grupo (logged_in) e uma melhor organização.
Mais detalhes na documentação:
[Documentação Next](https://nextjs.org/docs/app/building-your-application/routing)

### Pasta Components

#### Componentes separados por páginas

A pasta \_ui é onde ficam todos os componentes em comum entre várias páginas da aplicação, ex: (Botões, modais, etc), quase que como se fosse o nosso Design System.

E os demais componentes serão organizados em suas respectivas pastas (Ex: Componente LoginForm dentro da pasta login).

Pastas de componentes devem seguir o seguinte padrão:
